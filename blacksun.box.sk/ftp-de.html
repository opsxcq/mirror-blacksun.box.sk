    Was ist FTP und wofür brauche ich das?

     Die Abkürzung FTP (siehe Fußnote 1 unten) steht für File Transfer Protocol1. FTP Server lassen euch mit geringem Aufwand (wenn ihr dazu die nötigen Befugnisse habt) Dateien vom Server downloaden (eine Datei von Server emfangen) und uploaden (ein Datei zum Server senden). Eine Remote FTP Seite könnt ihr ähnlich durchsuchen und durchforsten wie ihr auf eurem Computer Dateien sucht und euch durch Verzeichnisse hangelt (natürlich habt ihr nicht Lese- und/oder Schreibzugriff auf jede Datei - manche Dateien könnt ihr nicht einmal sehen).


    FTP Befehle

     Im Folgenden findet ihr ein paar FTP Befehle, um mit den FTP Daemons7 zu kommunizieren. Verbindet euch mit Port2 21 und benutzt dann die folgenden Befehle (siehe Fußnote 2 unten) um mit FTP Server zu kommunizieren:

    cdVerzeichnis wchseln (auf dem Server)
    lcdlocales Verzeichnis wechseln (wenn ihr eine Datei sendet, wird der Pfad4 der Datei die ihr angegeben habt, der sein, den ihr mit lcd festgelegt habt)
    dir,lsVerzeichnisauflistung
    binaryModus zum Binären Transfer von Daten benutzen
    getDatei emfangen
    mgetmehrere Dateien emfangen
    putDatei senden
    mputmehrere Dateien senden
    pwdderzeitiges Verzeichnis (auf dem Server) ausdrucken

    Die Dumme-Bug-Ecke
     
     Ich fand das auf einer "elite" Web Site, die von einem Haufen "Elite Hackern" gemacht wurde.
    Sie sagten, das man, um ein FTP Server zu hacken, sich dort verbinden und folgende Befehle eintippen soll:

    quote user ftp
    quote cwd ~root
    quote pass ftp
    Was der sogenannte Hacker da versucht, ist eine Usernamen eingeben um in das System zu kommen, den User zum Root8 ändern und ein Passwort für den Usernamen eingeben.
    Das funtioniert aber nur bei wirklich schlecht konfigurierten FTP Servern (Der Autor meint, das funktioniere nicht bei allen FTP Servern. Naja, da habe ich Nachrichten für euch: das funktioniert überhaupt nicht, Punkt. Jedenfalls nicht, wenn wir hier über einen fünf Jahre alten Jungen sprechen, der gerade einen Computer bekommen, ein paar Knöpfe gedrückt und aus Versehen einen FTP Server aufgestellt hat.)


    Appendix A: the SYST command
     
     Gibt man den SYST Befehl ein während man zu einem FTP Server verbunden ist, bekommt man oft wertvolle informationen über das System, wie zum Beispiel das OS, welche Versionen und Information über den FTP Server.
    Verschafft euch irgendwie ein Verbindung zu einem FTP Server (durch benutzen eines Usernamens und eines Passwortes, welche ihr kennt oder durch einen anonymen Login - login: das anonyme Passwort ist eure-email-adreses@euer.isp. Ihr könntet auch die e-mail Adresse eines Anderen nehmen, der Sever prüft sie eigentlich nicht oder ähnliches) und gebt den SYST Befehl ein.


    Newbies-Ecke
     
    1.Protokol Eine Sammlung von Regeln und Regulierungen, änhlich einer Sprache. Wenn zwei Computer das gleiche Protokole kennen, benutzen sie es, um miteinander zu kommunizieren.
    2.Port (Für eine mehr technische Erklärung über was Port sind, scrollt zum etwas runter zum Ende der Erklärung von Ports) Ports sind wie Löcher, die bestimmten Dingen ermöglichen (in dem Fall sind es dann Daten) aus ihnen heraus- bzw. hereinzugelangen. Es gubt physische Ports und Software-Ports an euren Computern. Physische Ports sind die Slots an der Rückwand eures Computers, euers Monitor etc'. Software-Ports werden benutzt, um zu anderen Computern eine Verbindung herzustellen. Beispiel: Ich habe mir gerade einen neuen Computer gekauft und will ihn zu einem WebServer machen (Ich möchte, dass Leute ausgeählte Web-Seiten, Bilder, CGI- und Java Scripts oder Applets, Programme u.s.w. die auf meinem Computer sind, aufrufen können). Damit das möglich wird, muss ich eine WebServer Software installieren. Die WebServer Software öffntet einen Port an meinem Computer und gibt ihm den Namen "80". Dann wartet die Software auf eingehende Verbindungen an diesem Port. Wenn jetzt jemand seinen Internet Browser aufruft (Netscape, Lynx, Microsoft Explorer etc') und zu meiner Web-Seite surft, verbindet sein Browser sich mit meinem Computer an Port 80 und sendet dann HTTP Befehle, die mein WebServer dann versteht und die gewünschten Inhalte zurücksendet. Mein WebServer verarbeitet die eingehenden Daten ziehmlich schnell und sendet sie zurück zu dem Port, den der Browser auf dem Computer des Surfers geöffnet hat. Der Browser wird an dem Port auf geforderte Daten waretn (die HTML Seite, das Bild, das Programm etc'), die durch den Port kommen. Dann gibt es auch verschiedene Ports für verschiedene Daten (da kommen wir noch zu), damit die Daten nicht durcheinander kommen. Stellt euch vor, euer Browser bekommt Daten, die euer FTP Client bekommen sollte. Ich hoffe ihr habt so in etwa verstanden was ein Port ist.
    Jetzt gibt es da auch drei Arten von Ports: gut-bekannte Ports, registrierte Ports und dynamische/private Ports.
    Die bekannten Ports sind die von 0 bis 1023. Dies sind die Standart-Ports für verschiedene Dienste (ein WebServer ist ein Dienst, weil er auf eingehende Verbindungen von entfernten Computern wartet und dann etwas zurück sendet). Beispiel: der Standart-Port für WebServer ist 80. Wenn nicht, wie würde dann euer Browser wissen, zu welchem Port er verbinden soll
    Registrierte Ports sind die von 1024 bis 49151. Diese Ports sind reserviert für verschiedene Programme. Beispiel: ICQ (www.icq.com) reserviert einen Port und wartet auf eingehende Nachrichten.
    Die dynamischen und/oder privaten Ports sind die von 49152 bis 65535. Sie können von jedem für irgendetwas benutzt werden.

    "Technische Erklärung" - Um gleichzeitige Verbindungen zu dem TCP Modul zu gewährleisten, stellt TCP eine User Interface (UI) bereit, das sich Port nennt. Ports werden von Kernel benutzt um Netzwerk Aktivitäten zu identifizieren. Diese sind strickte Transport Schicht Dinge (IP's kümmert das weniger). Zusammen mit einer IP Adresse stellt ein TCP Portport einen Endpunkt für Netzwerkkommunikationen dar. Alle Internetverbindungen können in jedem Augenblick mit vier Nummern beschrieben werden: die Quellen-IP-Adresse mit dem Quellen-Port und ide Ziel-IP-Adresse mit dem Ziel-Port. Server sind an die 'gut-bekannten' Ports gebunden, so dass sie auf verschiedenen Systemen einen Standart-Port sein können . Zum Beispiel ist der Telnet Deamon an TCP Port 23, der FTP Daemon an TCP Port 21, der rlogin Daemon an TCP Port 513 u.s.w.

    Wichtiges über gut-bekannte Ports: Dienste (Daemons, die auf eingehende Verbindungen warten und Leuten irgendwie etwas nutzen) an diesen Ports können nur vom Root ausgeführt werden. So können untergeordnete User nicht irgendetwas an den wichtigen Port herumfummeln.
    3.Mirror-Site Eine Web-Site, die eine exakte Kopie der Originalen Web-Site ist und nur auf einem anderen Server ist. Mirror-Sites können benutzt werden, um Downloads/Uploads zu beschleunigen. Beispiel: Anstatt vom/auf den TUCOWS Haupt-WebServer herunterzuladen/hochzuladen, der irgendwo weit weg von meinem Haus steht, kann ich das gleiche von einem deutschen Mirror machen (Mirror-Site, die in Deutschland ist, wo ich wohne), wobei dann Downloads/Uploads schneller sind (meistens).
    4.Pfad UNIX Beispiel: wenn eine Datei /etc/passwd ist, ist der Pfad der Datei /etc.
    DOS/Windows Beispiel: wenn eine Datei c:\windows\win.exe ist, ist der Pfad der Datei c:\windows.
    Es gibt zwei Arten von Pfaden: ein kompletter Pfad und ein relativer Pfad.
    - Kompletter Pfad in DOS/Windows: wenn eine Datei in c:\program files\quickview plus\ ist, ist das ein kompletter Pfad.
    - Kompletter Pfad in UNIX: wenn eine Datei in /usr/local/sbin ist, ist das ein kompletter Pfad.
    - Relativer Pfad in DOS/Windows: wenn das derzeitige Verzeichnis (das Verzeichniss, wo ihr gerade seit) c:\windows ist, und die Zieldatei in c:\windows\temp ist, ist der relative Pfad zu dieser Datei "temp".
    - Relativer Pfad in UNIX: wenn das derzeitige Verzeichnis /usr/nobody ist, und die Zieldatei in /usr/nobody/public_html/cgi-bin ist, ist der relative Pfad zu dieser Datei "public_html/cgi-bin".
    5.Client / Server Programme Ein Client Programm ist ein Programm das Resources (Dinge) von einem anderen Programm/Computer benutzt.
    Ein Server Programm ist ein Programm das Resources an Client Programm liefert. Beispiel: Client=Netscape Navigator. Server=Apache version 1.6.6 (ein WebServer, also ein Programm das Leuten mit Internet Browsern ermöglicht bestimmte Web-Sites, Bilder, Dateien u.s.w. herunterzuladen).
    6.Information über Remote Hosts Der beste Weg um Informationen herauszufinden ist der, sich die Banner der Deamons7 anzuschauen. Daemon Banner sind kleine Stückchen an Information, die einige Daemons zurückliefern, wenn man verbunden ist, damit der verbindende Computer (der mit dem Deamon verbunden ist) weiß, wie er besser mit ihm kommuniziert. Versucht zu Port 80 (WebServer) zu verbinden und sendet Befehle wie GET und schaut dann den Banner an. Ihr könnt auch Sendmail an Port 25 vesuchen (siehe nächstes Tutorial), Telnet an Port 23, FTP an Port 21 oder womit auch immer.
    7.Daemon Ein Programm das auf eingehende Verbindungen von entferneten Computern an einem bestimmten Port2 wartet und dann mit diesen kommuniziert.
    8.Root Auch Superuser genannt, weil seine Erlaubnisse endlos sind. Seine UID (User ID Nummer, eine Identifikationsnummer die ein User auf einem UNIX System hat) und GID (Group ID. Ihr könnt Gruppen erstellen und diesen dann verschiedene Erlaubnisse erteilen. Beispiel: jeder der Berechnungsabteilung kann alle Dateien in diesem Verzeichnis lesen und ausführen, u.s.w.) sind immer 0 (außer bei sehr umgeänderten Boxes). Wenn ihr erstmal root seit, könnt ihr practisch alles an dem System herumexperimentieren.
    9.Core Dump Wenn ein Programm abstürtzt "schüttet" es den ganzen "Kern" (alle Informationen die verarbeitet wurden, die nicht auf Festplatte gespeichert sind. Also der ganze Kram eines Programms, das sich im RAM befindet) in eine temporäre Datei.
    10.DoS Denial of Service ("Dienstverweigerung"). Ein Nuker in Dummy-Sprache. Eine Art Attacke, die den Zielcomputer veranlasst einige/sämtliche Dienste zu Usern auf diesem Computer (auf entfernte User) zu verweigern. Beispiel: WinNuke (auch OOB genannt), der einfachste DoS der Welt. (Von Spikeman's DoS Seite) Dieses Denial of Service Programm beeinflusst Windows Clienten, indem eine "Out of Band" Ausnahmenachricht zu Port 139 gesendet wird, der nicht weiß, wie er damit umzugehen hat. Das ist der Standart Port, an dem die Windows Betriebssyseme auf Verbindungen warten. Benutzer von Win 3.11, Win 95 und Win NT sind mit dieser Attacke verwundbar. Dieser Programm ist eigentlich ein "Ärger-Programm", das viel im Internet herumgekommen ist. Es ist ein Umstand in Chat Räumen und IRC. Wenn man eure IP# nimmt und OOB Daten zu Port 139 sendet, können euch böswillige Benutzer vom Internet disconnecten (oft mit niedrigen Systemressourcen und Bluescreens) Einige von euch sind sicher schon einmal Opfer gewesen. Wenn euch das in Win 95 passiert, werdet ihr eine Window Fatal Error Message sehen, die ähnlich wie die folgende aussieht:
    Fatal exception 0E at 0028: in VxD MSTCP(01) + 000041AE.
    This was called from 0028: in VxD NDIS(01) + 00000D7C.
    Neustarten des System sollte alles wieder in Ordnung bringen
